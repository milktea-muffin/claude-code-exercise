# Task Planner

PRD と DESIGN を入力として、マイルストーン別のタスク一覧を作成する。

## 前提条件

docs/PRD.md と docs/DESIGN.md が存在すること。

## 作成手順

1. PRD から MVP 機能とマイルストーンを読み込む
2. DESIGN から技術スタック、DB テーブル、API エンドポイントを読み込む
3. assets/template.md のテンプレートに従って分解する

## 分解ルール

- 1タスク = 1ブランチ = 1PR のサイズ
- サイズ: S（半日以内）、M（1日）、L（2日以上は分割検討）
- 依存関係を明示（タスク番号で参照）
- 基盤タスクは最初のマイルストーンに配置
- DESIGN の全テーブルと全 API に対応するタスクを含める
- テスト、CI、デプロイ設定、README 更新も含める

## 品質チェック

- 依存関係にループがないか
- DESIGN の全コンポーネントがカバーされているか
- 各マイルストーンが独立してテスト可能か
