# PRD Writer

MVP 開発向けの PRD を作成するスキル。

## 情報収集

以下の情報を把握する。不足項目はまとめて1回で質問する。

### 必須

- 何を作るか（プロダクト概要）
- 誰が使うか（ターゲットユーザー）
- なぜ作るか（課題・動機）
- MVP に必須の機能

### 任意（なければ推測し「※推測」と注記）

- v2 以降の機能
- 技術的制約・好み
- スケジュール感
- チーム構成

## 作成手順

1. assets/template.md を読み込み、テンプレート構造を把握する
2. references/example.md を読み込み、記述の粒度とトーンを把握する
3. 収集した情報をテンプレートに当てはめて docs/PRD.md を作成する

## 品質基準（作成後セルフチェック）

- MVP 機能が 3-7 個に収まっているか（多すぎたら Later に移動）
- 矛盾する要件がないか
- 各機能に明確な説明があるか
- Screens が Features を網羅しているか
- Constraints にスコープ外が明記されているか
